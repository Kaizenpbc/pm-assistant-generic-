Filesrc/server/routes/integrations/mcpProxy.ts lines 38, 42, 86

#4 - Weather Cache Unbounded Growth

Filesrc/server/services/dataProviders/index.ts lines 41-42
CategoryMemory
StatusFIXED (782be3f)
DescriptionThe weather data cache (Map) grows without bound. pruneExpiredEntries() exists but is never called automatically. With unique coordinate combinations, memory usage grows indefinitely.
Fix AppliedAdded a MAX_CACHE_SIZE = 500 limit. On each cache insert, expired entries are pruned first. If still over limit, oldest entries are evicted (LRU by fetch time). Prevents unbounded growth.

Filesrc/server/services/EmbeddingService.ts lines 31-33
CategoryMemory

#11 - Resource Leveling O(n^2) Without Timeout

Filesrc/server/services/ResourceLevelingService.ts lines 226-280
CategoryPerformance
StatusFIXED (782be3f)
DescriptionThe leveling algorithm has nested loops: for each non-critical task, iterate through its float range, and for each delay attempt, iterate through all task days. With 500+ tasks, this is O(n^2) with no timeout or iteration limit.
Fix AppliedAdded a MAX_LEVELING_ITERATIONS = 50000 counter. The inner loops increment the counter, and leveling stops gracefully when the limit is reached. Logs a warning when truncated. Prevents runaway computation on large schedules.

#12 - Monte Carlo Unbounded Memory

Filesrc/server/services/MonteCarloService.ts lines 71-107
CategoryPerformance / Memory
StatusFIXED (782be3f)
DescriptionFor 10,000 iterations with N tasks, sampledDurationsPerTask stores 10,000 x N numbers. For a schedule with 1000 tasks, this is ~40MB of memory per simulation run. No cap on iterations.
Fix AppliedAdded MAX_ITERATIONS = 50000 cap with a clamp. If the user requests more than 50K iterations, it silently caps at 50K. This bounds worst-case memory while preserving statistical accuracy.

Appendix: Methodology

  1. Security - CORS, secrets, SQL injection, auth, SSRF, XSS, path traversal, JWT, file upload, rate limiting

  3. Error Handling - Silent catches, missing error handling, unhandled rejections, frontend error states, crash-prone patterns
  4. Code Quality - Dead code, duplicates, type safety, console.log, magic numbers, config inconsistencies
  5. Frontend & API Contracts - Data mismatches, placeholders, loading/error states, dark mode, accessibility, responsive design
